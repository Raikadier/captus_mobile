Cómo una pantalla consume un provider

// 1. La pantalla extiende ConsumerWidget (sin setState)
class HomeScreen extends ConsumerWidget {
  @override
  Widget build(BuildContext ctx, WidgetRef ref) {

    // 2. ref.watch → reconstruye cuando cambia
    final tasksAsync = ref.watch(tasksProvider);

    // 3. .when() maneja loading / error / data automáticamente
    return tasksAsync.when(
      loading: () => TaskListShimmer(),
      error:   (e, _) => ErrorState(message: friendlyError(e)),
      data:    (tasks) => TaskList(tasks: tasks),
    );
  }
}
Módulo IA

Pipeline del Asistente IA

El asistente no es un chatbot genérico. Clasifica la intención del mensaje y ejecuta herramientas reales sobre la base de datos del usuario antes de responder.

Arquitectura del Pipeline — 6 archivos en src/ai/

1
routerAgent.js — Clasificación de intención
Recibe el mensaje del usuario. Llama a Gemini Flash (modelo rápido y barato) con el prompt del sistema para clasificar el intent. Intents posibles: task note course assignment general
2
context.js — RAG: contexto dinámico
Según el intent clasificado, hace queries específicas a Supabase para traer los datos relevantes del usuario (tareas pendientes, cursos activos, notas recientes, etc.) y los inyecta en el prompt.
3
orchestrator.js — Ejecución de herramientas
Con el contexto inyectado, llama a Gemini Pro. Si el modelo decide ejecutar una herramienta (crear tarea, buscar curso, etc.), el orchestrator la ejecuta en la BD y retorna el resultado al modelo para que genere la respuesta final.
4
toolRegistry.js — 20+ herramientas registradas
Cada herramienta tiene un schema JSON que el modelo usa para llamarla. Ejemplos: create_task, list_pending_tasks, get_course_detail, create_note, mark_task_complete.
5
Respuesta estructurada al cliente
El backend retorna { result, steps[], actionPerformed, conversationId }. El cliente Flutter muestra los steps expandibles y persiste la conversación.

Ejemplo de herramienta en toolRegistry.js

// Cada herramienta: schema para el modelo + handler que ejecuta la acción
{
  name: 'create_task',
  description: 'Crea una nueva tarea para el usuario',
  parameters: {
    title: { type: 'string', required: true },
    due_date: { type: 'string', format: 'ISO8601' },
    priority: { type: 'string', enum: ['high', 'medium', 'low'] }
  },
  handler: async (args, userId) => {
    const task = await taskService.create({ ...args, userId });
    return { success: true, task };
  }
}

Código Clave

1. Auth guard en GoRouter

Redirige al login si no hay sesión, al rol correcto si la hay.

redirect: (context, state) {
  final auth = ref.read(authProvider);
  final isAuth = auth.asData?.value.isAuthenticated;
  if (isAuth != true) return '/login';

  final role = ref.read(userRoleProvider);
  if (role == 'admin') return '/admin/dashboard';
  if (role == 'superadmin') return '/superadmin';
  return null; // permite continuar
}

2. ApiClient — Interceptor JWT

Inyecta el token de Supabase en cada petición automáticamente.

_dio.interceptors.add(InterceptorsWrapper(
  onRequest: (options, handler) {
    final token = Supabase.instance
      .client.auth.currentSession?.accessToken;
    if (token != null) {
      options.headers['Authorization']
        = 'Bearer $token';
    }
    return handler.next(options);
  },
));

3. Validate Middleware — Zod

Un middleware genérico que valida el body con cualquier schema Zod.

// Un middleware reutilizable para cualquier endpoint
export const validate = (schema) => (req, res, next) => {
  const result = schema.safeParse(req.body);
  if (!result.success) {
    return res.status(400).json({
      error: { code: 'VALIDATION_ERROR',
               details: result.error.issues }
    });
  }
  req.body = result.data; // datos ya parseados
  return next();
};

4. friendlyError() — Errores en español

Convierte cualquier excepción en un mensaje legible para el usuario.

String friendlyError(Object e) {
  if (e is DioException) {
    switch (e.type) {
      case DioExceptionType.connectionTimeout:
        return 'Conexión tardó demasiado.';
      case DioExceptionType.badResponse:
        return _fromStatusCode(e.response?.statusCode);
    }
  }
  // Nunca expone stack traces al usuario
  return 'Algo salió mal. Intenta de nuevo.';
}

5. AiChatNotifier — send()

La operación más compleja del cliente: envía mensaje, maneja cancelación y parsea la respuesta estructurada.

Future<void> send(String text) async {
  _cancelToken = CancelToken();
  state = state.copyWith(isLoading: true);

  final res = await ApiClient.instance.post(
    '/ai/chat',
    data: { 'message': text },
    cancelToken: _cancelToken, // ← cancelable
  );

  final steps = (res.data['steps'] as List)
    .map(AiStep.fromJson).toList();

  state = state.copyWith(
    messages: [...state.messages, botMsg],
    steps: steps, isLoading: false,
  );
}

6. connectivityProvider — Stream reactivo

Detecta cambios de red en tiempo real. El banner offline se actualiza automáticamente.

// Emite true/false cada vez que cambia la conectividad
final connectivityProvider =
  StreamProvider<bool>((ref) {
    return Connectivity()
      .onConnectivityChanged
      .map((results) => results.any(
        (r) => r != ConnectivityResult.none
      ));
  });

// El widget OfflineBanner lo lee sin setState()

Seguridad

Capas de Seguridad

🛡️ Backend

Helmet — 14 headers de seguridad HTTP, Content Security Policy
Rate Limiting — 200 req/min general, 30 req/min en /ai/chat
JWT Supabase — cada endpoint verifica el token antes de ejecutar
Zod validation — body sanitizado y tipado antes del handler
Payload limit — máximo 50KB por request (anti DDoS)
Sentry — captura y alerta de errores 5xx en producción
Errores genéricos — en producción, los 500 retornan mensaje genérico (no expone stack)

📱 Mobile

flutter_secure_storage — tokens JWT en el keychain del sistema operativo
ProGuard — minificación y ofuscación en release build
usesCleartextTraffic: false — solo HTTPS en Android
Inactivity logout — cierre automático de sesión por inactividad
Crashlytics — captura errores en Flutter y errores nativos
friendlyError() — nunca se exponen mensajes técnicos al usuario
CancelToken — peticiones IA cancelables, evita respuestas huérfanas
